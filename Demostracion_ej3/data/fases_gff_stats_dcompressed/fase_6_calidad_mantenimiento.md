# Fase 6: Calidad del código y mantenimiento

## Objetivo
Mantener el código robusto, legible y fácil de actualizar, siguiendo buenas prácticas de bioinformática y desarrollo reproducible.

## Acciones
- Formato consistente según PEP8.
- Nombres de variables y funciones claros y significativos.
- Modularidad: cada función realiza una única tarea.
- Posibles extensiones futuras:
  - Soporte de archivos GTF.
  - Procesamiento de múltiples archivos GFF.
  - Visualización de estadísticas.
- Control de versiones:
  - Uso de Git/GitHub para versionado del código y datos de prueba.
- Logging opcional:
  - Advertencias sobre líneas ignoradas o errores en el formato.
- Separación de datos de prueba y datos reales.

## Buenas prácticas bioinformáticas
- Versionado de scripts y datasets de prueba.
- Mantener trazabilidad y reproducibilidad.
- Documentar cambios y nuevas funcionalidades.
- Preparar el código para integración en pipelines bioinformáticos.
